Digital Park

Semana a semana

Sebastián Castillo Mayo 12, 2020 0 Comentarios

Semana 1

En la primera semana de trabajo, desde la universidad sabíamos que íbamos a hacer, una plataforma tipo red social, en la cual los usuarios reportan sus acciones a favor del medio ambiente y las compartieran con sus amigos y familiares, y que de cierta forma, se llame la atención de las personas y se haga un llamado de consciencia. Luego de la crisis del Covid-19, pasaron unas semanas en las cuales el equipo no sabia que hacer, que iniciativas tomar, como trabajar y cuál era la mejor forma para proseguir en este proyecto. La solución, hablar con la docente que nos encargó este proyecto. La forma de proceder se decidió esto: Para trabajar, los miembros del grupo se dividirán de acuerdo a las necesidades que el proyecto necesitará para entregar un proyecto de calidad, uno de los integrantes se iba a encargar de la parte de la programación, aquí, se pide investigar nuevos métodos y librerías con el fin de enseñar el producto, y así, un motor que sea capaz de lograr los requerimientos para el proyecto y la propuesta dada. Otro de los miembros del equipo, se encargará del contenido multimedia, en producción y post-producción de productos necesarios para mostrar nuestra propuesta como algo innovador y que rompa con esquemas. Aquí, el miembro del equipo encargado será el encargado de realizar videos, imagenes, y otra función de este miembro, será la creación del blog, con vista a sus capacidades, este miembro tiene conocimientos de css y html, con los cuales, la página web a crear y que es uno de los requerimientos para este proyecto. El último miembro del equipo, será el encargado del contenido, este miembro hará los respectivos textos necesarios para la página, la aplicación y el behance, pruebas de usuario y demás.

Semana 2

Para esta semana, el miembro encargado de la programación trajo a colación el tema de una motor gráfico y que se podía manejar en el lenguaje de JavaScript, un lenguaje que todos los miembros del equipo manejan. Este miembro le pidió a los demás que descargaron el programa Unity, un programa muy conocido para crear juegos en 3D y que era fácil de usar, por la cantidad de tutoriales que había sobre este programa. Desafortunadamente, los miembros no pudieron descargar este programa, bien sea por la memoria que se requería para este programa, o por los recursos que necesitaba el c para descargar un programa que necesitaba de tanto. Al final, fue descartada esta idea, y todo el grupo tuvo que investigar acerca de librerías o programas que se ajustaran a nuestras necesidades. Al final de semana, y con una pre-entrega en la puerta de la esquina, uno de los miembros lanzó un rayo de esperanza, una librería llamada Three.js, una librería que funcionaba con un editor de código que los miembros del equipo conocían muy bien, el Visual Studio Code, esta librería se importaba a un archivo js y se podía empezar a trabajar de forma ordinaria. Además, para facilidad del grupo, Three.js tiene muchas plantillas y proyectos realizados por otras personas, con el fin de ayudar a personas que no tienen mucho conocimiento del tema de programación y modelado. Para la entrega de esa semana, se entregó un modelo de baja calidad de nuestra idea, un árbol en 3D, el usuario podía manejar la camara como le plazca.

Semana 3

Para esta semana, el problema es el siguiente, ya creamos un árbol funciona en la aplicación, pero, ¿Cómo crear varios árboles de acuerdo a las acciones que realizan nuestros usuario? Esto no preocupaba mucho a los miembros del grupo, la respuesta podría parecer obvia, una base de datos conectada a Firebase, una base de datos vista por los integrantes del grupo el semestre pasado. Firebase nos permite crear variables y actualizarlas de forma rápida cuando se ha alcanzado un estado específico o se ha modificado algo en una variable. Aparte, como se pensaba crear una red social en base a esta idea de cuidar el medio ambiente, en Firebase se pueden llenar arreglos con nombres, correos y estados de una variable en específico, justo lo que necesitábamos para esto. Al realizar pruebas, la base de datos respondía a un número y el registro de un usuario, además de recordar la sesión de un usuario que se ha registrado anteriormente. Al guardar esto, se guarda en una variable el número de árboles que se tiene para dibujar en el mundo virtual.

Semana 4

Para esta semana, la base de datos funcionaba y era momento de solucionar un problema bastante grande, se quería que el usuario tomara foto de sus acciones y las subiera a la aplicación, y luego, esto sería validado por la aplicación, y el número de árboles aumentaba. Esto terminó siendo un dolor de cabeza para el equipo de trabajo, la forma de crear una validacion de ese nivel y con tan pocas semanas para hacer algo con una inteligencia artificial integrada, sonaba casi imposible. Se manifestó esta preocupación a nuestra docente, y las respuesta de ella fue la que dió un nuevo horizonte a todo el proyecto. La idea de una inteligencia artificial, en definitiva era muy difícil para integrar, y la idea fue la siguiente: Crear un cuestionario en el que las personas resolvieran preguntas, y el número de árboles se diera por la cantidad de puntos que se obtuvieron en esta encuesta. Esto, cambió por completo el panorama del equipo de trabajo, a partir de esto, el proyecto ya no sonaba tan difícil y asficciante como antes, la idea de integrar un sistema de puntos y crear preguntas coherentes a nuestro tema, no sonaba difícil. Gracias a esta idea, partimos con el hecho de descartar gran parte de lo que teníamos planeado, pero, todo esto era con el fin de entregar un producto de buena calidad.

Semana 5

Para la semana final de trabajo, se tenía que tener todo en lo que consistia la entrega, se continuó trabajando en las modificaciones del código, y aunque hubo problemas en la base de datos, problemas que nunca habían aparecido, como una conexión inválida, o que el puerto no estaba disponible para las operaciones, todos estos problemas fueron resueltos. Se tenía que realizar una página web, un blog, en la que mostrasemos el proceso de nuestro proyecto, con bocetos, ideas, explicacion y demás, esta pagina fue realizada en pocos dias, ya que los miembros del equipo repartieron las actividades de forma efectiva, y cada uno realizaba un html distinto, manteniendo claramente un estilo gráfico uniforme, además de utilizar el atomic design, esto nos permitió partir desde pequeños átomos de diseño y crear moléculas y partículas más grandes, para que cada página se viera consistente con las demás. Aparte, para un proyecto de diseño se tenía que crear un Behance, en donde se muestra nuestro producto, que en este semana lo hemos llamado Digital Park, en este Behance, explicamos y mostramos más a detalle algunos datos que estuvieron detras de camaras de este proyecto, cuidando algunos detalles, ya que nuestro Behance, es la cara, la portada de nuestro proyecto. Para finalizar esta entrega, era necesario un Trailer, algo con un alcance y una magnitud tanta, que las personas lo vieran y sintieran la necesidad de usar nuestra aplicación. Es por esto que se ha creado un teaser trailer, mostrando que es Digital park y el alcance que este puede llegar a tener, el potencial que tiene nuestro producto.
